# Legal Review Checklist

รายการตรวจสอบด้วยมือสำหรับผู้ดูแลเนื้อหากฎหมายการศึกษา — ใช้คู่กับ `npm run check:content` และ GitHub Actions

## แหล่งข้อมูลและ metadata

- [ ] `source_url` ชี้ไปแหล่งทางการที่เปิดอ่านได้ (ราชกิจจานุเบกษา กฤษฎีกา หรือหน่วยงานที่เกี่ยวข้อง)
- [ ] `official_source` ระบุชื่อแหล่งถูกต้อง ไม่ใช่ placeholder “ต้องตรวจสอบจากแหล่งทางการ”
- [ ] `last_checked` อัปเดตเป็นวันที่ตรวจล่าสุดจริง
- [ ] `status` ไม่กล่าวเกินจริง (เช่น ไม่เขียน “บังคับใช้” ถ้ายังเป็นร่าง)
- [ ] `year`, `law_type`, `agency`, `category` สอดคล้องกับฉบับจริง

## เนื้อหาสรุป

- [ ] สรุป (`summary` / `points`) ไม่เปลี่ยนความหมายกฎหมาย
- [ ] ไม่ใช้สรุปแทนการอ่านฉบับเต็มในบริบทที่ต้องอ้างอิงทางกฎหมาย
- [ ] มี disclaimer / หมายเหตุว่าเป็นการสรุปเพื่อการเรียนรู้ (เทมเพลตมีให้แล้ว)
- [ ] ไม่มี placeholder (`TODO`, `TBD`, ข้อความชั่วคราว)

## การเชื่อมโยงและเครื่องมือ

- [ ] ลิงก์ใน Markdown และ JSON เปิดได้ (ดู workflow Link Check)
- [ ] หน้า taxonomy / ตัวกรองยังแสดงถูกต้องหลังแก้ metadata
- [ ] Pagefind ยัง build ได้ (`npm run build:search`)

## แบบทดสอบ (ถ้าเกี่ยวข้อง)

- [ ] คำอธิบายเฉลยใน `data/quizzes.yaml` ปลอดภัย ไม่วินิจฉัยเฉพาะกรณี
- [ ] `related_law_id` ชี้ไปกฎหมายที่เกี่ยวข้องจริง
- [ ] ไม่สร้างข้อเท็จจริงทางกฎหมายใหม่ในคำถามหรือเฉลย

## ก่อน merge / deploy

- [ ] `npm run check:content` ผ่าน (ไม่มี critical)
- [ ] `npm run build:search` ผ่าน
- [ ] ตรวจ warnings ที่เหลือและตัดสินใจว่าจะแก้หรือยอมรับชั่วคราว
- [ ] งานที่เกี่ยวกับความถูกต้องทางกฎหมาย — **ตรวจโดยผู้มีความรู้ ไม่ใช่สคริปต์**
