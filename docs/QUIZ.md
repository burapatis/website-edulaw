# แบบทดสอบความรู้ (Learning Quiz)

แบบทดสอบเป็นเครื่องมือเรียนรู้แบบ static — ไม่มี login ไม่มีฐานข้อมูล ไม่เก็บข้อมูลส่วนบุคคล

> **เพื่อการเรียนรู้เท่านั้น** — แบบทดสอบนี้ใช้ทบทวนความเข้าใจเบื้องต้น ไม่ใช่การวินิจฉัยหรือให้คำปรึกษาข้อกฎหมายเฉพาะกรณี และไม่ใช้แทนแหล่งข้อมูลทางการ

## ภาพรวมชุดข้อสอบ

- มีคำถามทั้งหมด **100 ข้อ**
- แบ่งเป็น **5 หมวด** หมวดละ **20 ข้อ**
- โหมด **แบบทดสอบรวม (สุ่ม 20 ข้อจากทุกหมวด)** จะสุ่มเลือกเพียง 20 ข้อจาก 100 ข้อ ไม่แสดงครบทุกข้อ และสุ่มใหม่ทุกครั้งที่เริ่ม/ทำใหม่ โดยไม่ซ้ำข้อภายในรอบเดียวกัน
- คำถามแต่ละข้อระบุ **ระดับการวัดตาม Bloom (`bloom_level`)**

5 หมวด ได้แก่ `constitution`, `national`, `personnel`, `admin-higher`, `child-institution`

## ที่เก็บข้อมูลคำถาม

| ไฟล์ | หน้าที่ |
|------|---------|
| `data/quizzes.yaml` | **แหล่งข้อมูลหลัก** — หมวดและคำถามทั้งหมด |
| `layouts/quiz/list.html` | เทมเพลตหน้า `/quiz/` ฝัง JSON จาก Hugo data |
| `static/js/quiz.js` | ตัวกรอง (หมวด/กลุ่มผู้ใช้/ความยาก/Bloom) ทำข้อสอบ คะแนน เฉลย แสดงระดับ Bloom ลิงก์อ่านต่อ |
| `scripts/check-quiz-data.js` | สคริปต์ตรวจสอบข้อมูลแบบทดสอบ (`npm run check:quiz`) |
| `static/js/quiz-data.js` | สำรองอ้างอิง (เวอร์ชัน JS เดิม) — หน้าเว็บไม่โหลดแล้ว |

## โครงสร้างข้อมูล

```yaml
categories:
  - key: national
    title: "การจัดการศึกษาแห่งชาติ"
    description: "..."
    default_difficulty: "พื้นฐาน"
    default_audience: "ครูและบุคลากรทางการศึกษา"
    related_law_id: nea-2542
    related_law_title: "พระราชบัญญัติการศึกษาแห่งชาติ พ.ศ. 2542"

questions:
  - id: q001
    category: national          # ตรงกับ categories[].key
    audience: "ครูและบุคลากรทางการศึกษา"
    difficulty: "พื้นฐาน"
    bloom_level: "จำ"            # ระดับการวัดตาม Bloom (ดูด้านล่าง)
    question: "ข้อความคำถาม"
    choices:
      - "ตัวเลือก ก"
      - "ตัวเลือก ข"
      - "ตัวเลือก ค"
      - "ตัวเลือก ง"
    answer: 0                     # ดัชนี 0–3 (ก=0, ข=1, ค=2, ง=3)
    explanation: "คำอธิบายเดิมจากชุดข้อสอบ"
    related_law_id: nea-2542      # id ใน data/laws.json
    related_law_title: "ชื่อกฎหมายสำหรับลิงก์อ่านต่อ"
    related_topics: []
```

## ระดับการวัดตาม Bloom (`bloom_level`)

`bloom_level` บอกว่าคำถามนั้นวัดการเรียนรู้ระดับใดตามทฤษฎี Bloom's Taxonomy (ฉบับปรับปรุง) เรียงจากระดับล่างไปบน:

| ค่า | ความหมาย | ตัวอย่างคำถาม |
|-----|----------|----------------|
| `จำ` | จดจำข้อเท็จจริง ชื่อ ปี ตัวเลข | "พ.ร.บ. ประกาศใช้ปีใด" |
| `เข้าใจ` | อธิบายความหมาย/หลักการ | "ระบบทวิภาคีหมายถึงอะไร" |
| `นำไปใช้` | นำหลักไปใช้กับสถานการณ์ | "หากผู้ปกครองไม่ส่งเด็กเข้าเรียน จะเกิดผลใด" |
| `วิเคราะห์` | แยกแยะ/เปรียบเทียบ/หา "ข้อใดไม่ใช่" | "ข้อใดไม่ใช่องค์กรหลักของกระทรวง" |
| `ประเมิน` | ตัดสินคุณค่า (ปัจจุบันยังไม่ใช้ เพื่อเลี่ยงคำถามเชิงความเห็น) | — |
| `สร้างสรรค์` | ออกแบบ/สร้างใหม่ (ปัจจุบันยังไม่ใช้) | — |

- ระบบจะ **แสดงระดับ Bloom** ใต้หัวข้อ "คำอธิบาย" เมื่อผู้ใช้ตอบข้อนั้นแล้ว เช่น `ระดับการวัดตาม Bloom: วิเคราะห์`
- หน้าเลือกชุดมี **ตัวกรอง "ระดับการวัดของ Bloom"** ซึ่งสร้างตัวเลือกจากระดับที่มีจริงในข้อมูล โดยเรียงตามลำดับทฤษฎี Bloom
- ค่า `bloom_level` ที่รองรับ: `จำ`, `เข้าใจ`, `นำไปใช้`, `วิเคราะห์`, `ประเมิน`, `สร้างสรรค์`

## วิธีเพิ่มคำถามใหม่

1. เปิด `data/quizzes.yaml`
2. เพิ่มรายการใน `questions:` ด้วย `id` ที่ไม่ซ้ำ (เช่น `q101`) — ใช้การเยื้อง 2 ช่องว่างสำหรับ `- id:` และ 4 ช่องว่างสำหรับฟิลด์ย่อย
3. ตั้ง `category` ให้ตรงกับ `categories[].key` ที่มีอยู่
4. ใส่ `bloom_level` ให้ตรงกับชุดที่รองรับ (`จำ`/`เข้าใจ`/`นำไปใช้`/`วิเคราะห์`/`ประเมิน`/`สร้างสรรค์`)
5. ใส่ `choices` ให้ครบ **4 ตัวเลือก** เสมอ
6. ตั้ง `answer` เป็นเลขดัชนีของตัวเลือกที่ถูก (เริ่มที่ 0)
7. ใส่ `explanation` ทุกข้อ และอ้างอิงเฉพาะข้อเท็จจริงจากกฎหมาย ไม่ให้คำปรึกษาเฉพาะกรณี
8. ใส่ `related_law_id` ตาม `id` ใน `data/laws.json` เพื่อลิงก์ไปหน้ารายละเอียดกฎหมาย
9. รัน `npm run check:quiz` เพื่อตรวจความถูกต้อง แล้ว `hugo server -D` ทดสอบที่ `/quiz/`

> หมายเหตุ: ระบบกำหนดให้แต่ละหมวดมี 20 ข้อ และรวม 100 ข้อ หากเพิ่ม/ลดจำนวน ต้องปรับ `EXPECTED_*` ใน `scripts/check-quiz-data.js` ให้สอดคล้องด้วย

## ดัชนีคำตอบ (`answer`)

- `0` = ตัวเลือกแรก (ก)
- `1` = ตัวเลือกที่สอง (ข)
- `2` = ตัวเลือกที่สาม (ค)
- `3` = ตัวเลือกที่สี่ (ง)

## ลิงก์กฎหมายที่เกี่ยวข้อง

- `related_law_id` ใช้สร้าง URL: `/laws/{id}/` (ผ่าน `data-laws-base` ในเทมเพลต)
- ถ้าไม่ระบุ ระบบจะลิงก์ไปคลังกฎหมายทั่วไป
- อย่าใส่ URL เต็มใน YAML — ใช้ `related_law_id` ให้สอดคล้องกับ Hugo baseURL

## แปลงจากไฟล์ JS เดิม

หากแก้ `static/js/quiz-data.js` ชั่วคราว สามารถรัน:

```bash
node scripts/convert-quiz-to-yaml.mjs
```

เพื่อสร้าง `data/quizzes.yaml` ใหม่ (ระวังทับการแก้ไขใน YAML)

## ทดสอบในเครื่อง

```bash
hugo server -D
# หรือ
npm run dev
```

เปิด http://localhost:1313/website-edulaw/quiz/

ตรวจสอบ:

- เลือกหมวด / กลุ่มผู้ใช้ / ระดับความยาก / ระดับ Bloom
- หมวดเฉพาะแสดง 20 ข้อ และโหมดรวมสุ่มแสดงเพียง 20 ข้อ (เปลี่ยนข้อ/ลำดับเมื่อทำใหม่)
- กด **เริ่มทำแบบทดสอบ**
- ตอบแล้วเห็น **ถูกต้อง** / **ยังไม่ถูกต้อง** + คำอธิบาย + ป้าย **ระดับการวัดตาม Bloom** + **อ่านต่อ**
- ดูคะแนนรวมและทำใหม่อีกครั้ง

ตรวจสอบข้อมูลและ build production:

```bash
npm run check:quiz     # ตรวจ 100 ข้อ / 5 หมวด / 20 ข้อต่อหมวด / id / answer / bloom_level / explanation
npm run build:search   # Hugo minify + Pagefind
```

`check:quiz` ถูกรวมไว้ใน `npm run release:check` และ GitHub Actions (Quality Check) ด้วย

## ข้อจำกัด

- คำอธิบายมาจากชุดข้อสอบเดิม — ควรให้ผู้เชี่ยวชาญทบทวนก่อนใช้อ้างอิงอย่างเป็นทางการ
- Pagefind ไม่ index คำถามใน JS/YAML โดยตรง (มีเฉพาะข้อความแนะนำบนหน้า `/quiz/`)
- ไม่บันทึกประวัติการทำข้อสอบของผู้ใช้
