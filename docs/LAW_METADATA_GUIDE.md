# คู่มือ Metadata กฎหมาย (Front Matter / JSON)

อธิบายฟิลด์ metadata สำหรับหน้ากฎหมายใน Education Law Hub — ใช้ได้ทั้ง `data/laws.json` และ `content/laws/*.md`

## สรุปความจำเป็น

| ฟิลด์ | จำเป็น | Critical ถ้าขาด |
|--------|--------|-----------------|
| title | ใช่ | ใช่ |
| source_url | ใช่ (หลัง derive) | ใช่ |
| official_source | ใช่ | ใช่ (ว่างจริง) |
| last_checked | ใช่ | ใช่ (ใน .md) |
| short_title | แนะนำ | ไม่ |
| description | แนะนำ | ไม่ |
| law_type, year, category, agency | แนะนำ | ไม่ |
| audiences, tags | แนะนำ | ไม่ |
| draft | ใช่ (ต้องเป็น false) | ถ้า true |

---

## title

**ความหมาย:** ชื่อเต็มของกฎหมายหรือเอกสารตามที่ปรากฏในแหล่งทางการ

**ตัวอย่าง:** `พระราชบัญญัติการศึกษาแห่งชาติ พ.ศ. 2542 และที่แก้ไขเพิ่มเติม`

**ข้อผิดพลาดที่พบบ่อย:** ย่อชื่อจนเปลี่ยนความหมาย; ใส่สถานะ “ฉบับล่าสุด” ในชื่อโดยไม่ตรวจ

**ผลกระทบ:** หัวข้อหน้า, SEO title, metadata card, Pagefind

---

## short_title

**ความหมาย:** ชื่อย่อสำหรับการ์ดและการอ้างอิงสั้น

**ตัวอย่าง:** `พ.ร.บ.การศึกษาแห่งชาติ`

**ข้อผิดพลาดที่พบบ่อย:** ว่าง (ระบบจะ derive จาก title — ได้ warning)

**ผลกระทบ:** law cards, metadata card

---

## description

**ความหมาย:** คำอธิบายสั้นสำหรับ SEO และการแชร์

**ตัวอย่าง:** `สรุปและข้อมูลเบื้องต้นเกี่ยวกับ ... พร้อมลิงก์แหล่งข้อมูลทางการ`

**หมายเหตุ:** หน้าจาก `laws.json` สร้าง `description` อัตโนมัติใน `_content.gotmpl` หากไม่ระบุใน JSON

**ข้อผิดพลาดที่พบบ่อย:** อ้างว่า “ครบถ้วน” หรือ “ฉบับล่าสุดแน่นอน”

**ผลกระทบ:** SEO, Open Graph

---

## law_type / type

**ความหมาย:** ประเภทเอกสารทางกฎหมาย

**ตัวอย่าง:** `พระราชบัญญัติ`, `กฎกระทรวง`, `ระเบียบ`, `ประกาศ`

**ข้อผิดพลาดที่พบบ่อย:** ใส่ “กฎหมาย” ทั่วไป; สับสนระหว่าง `type` กับ `law_type` (ใช้ค่าเดียวกันได้)

**ผลกระทบ:** metadata card, taxonomy `/lawtypes/`, ตัวกรอง

---

## year

**ความหมาย:** ปีที่ประกาศหรือปีที่เกี่ยวข้องกับฉบับหลัก

**ตัวอย่าง:** `พ.ศ. 2542`

**ข้อผิดพลาดที่พบบ่อย:** ใส่ปีแก้ไขล่าสุดโดยไม่ระบุในชื่อ/หมายเหตุ

**ผลกระทบ:** law cards, การเรียงลำดับ

---

## category

**ความหมาย:** key หมวดหลักในระบบ (ดู `categories` ใน `data/laws.json`)

**ตัวอย่าง:** `national`, `personnel`, `child`

**ข้อผิดพลาดที่พบบ่อย:** ใช้ label ภาษาไทยแทน key

**ผลกระทบ:** สีการ์ด, ตัวกรอง `/laws/?cat=`, taxonomy

---

## categories

**ความหมาย:** อาร์เรย์หมวด (รองรับหลายหมวด)

**ตัวอย่าง:** `["national", "personnel"]`

**หมายเหตุ:** ถ้าว่าง ระบบใช้ `[category]` อัตโนมัติ

**ผลกระทบ:** taxonomy `/categories/`

---

## agency

**ความหมาย:** หน่วยงานหลักที่เกี่ยวข้อง

**ตัวอย่าง:** `สำนักงานเลขาธิการสภาการศึกษา (สกศ.)`

**ผลกระทบ:** metadata card, ตัวกรองหน่วยงาน, SEO description

---

## agencies

**ความหมาย:** taxonomy หน่วยงาน (มัก derive จาก `agency` ใน gotmpl)

**ตัวอย่าง:** ใน `.md` อาจระบุหลายหน่วยงานถ้า Hugo taxonomy รองรับ

**ผลกระทบ:** `/agencies/`

---

## status

**ความหมาย:** สถานะข้อมูล/สถานะกฎหมายในเว็บไซต์

**ตัวอย่าง:** `บังคับใช้`, `ร่าง`, `เลิกใช้บางส่วน`

**ข้อผิดพลาดที่พบบ่อย:** เขียน “บังคับใช้” ทั้งที่ยังเป็นร่าง; ใช้คำคลุมเครือ “ประกาศใช้”

**ผลกระทบ:** metadata card, badges บนหน้ารายละเอียด

---

## source_url

**ความหมาย:** URL แหล่งทางการหลักสำหรับเปิดอ่านฉบับ

**ตัวอย่าง:** `https://ratchakitcha.soc.go.th/...`

**ข้อผิดพลาดที่พบบ่อย:** เดา URL; ลิงก์เสีย; ลิงก์สรุปแทนฉบับเต็มโดยไม่อธิบาย

**ผลกระทบ:** ปุ่ม “เปิดแหล่งข้อมูลทางการ”, **critical** ถ้าว่าง

---

## official_source

**ความหมาย:** ชื่อแหล่งอ้างอิงทางการ

**ตัวอย่าง:** `ราชกิจจานุเบกษา`, `สำนักงานคณะกรรมการกฤษฎีกา`

**Placeholder:** `ต้องตรวจสอบจากแหล่งทางการ` — ใช้ได้ชั่วคราวแต่ได้ **warning**

**ผลกระทบ:** metadata card

---

## last_checked

**ความหมาย:** วันที่ทบทวนข้อมูลล่าสุด (`YYYY-MM-DD`)

**ตัวอย่าง:** `2026-06-24`

**หมายเหตุ:** ใน JSON หากไม่ระบุ `_content.gotmpl` ใช้ `$structuralReviewDate`

**ผลกระทบ:** metadata card; warning ถ้าเก่ากว่า 180 วัน (โหมด `--full`)

---

## audiences

**ความหมาย:** กลุ่มผู้ใช้ที่เหมาะกับเนื้อหา

**ตัวอย่าง:**

```yaml
audiences:
  - ครูและบุคลากรทางการศึกษา
  - นักศึกษา
```

**Default (ถ้าไม่ระบุ):** ครู, นักศึกษา, ผู้สนใจทั่วไป

**ผลกระทบ:** `/audiences/`, ตัวกรอง, learning paths

---

## tags

**ความหมาย:** คำสำคัญสำหรับค้นหาและจัดกลุ่ม

**ตัวอย่าง:** `["วิทยฐานะ", "ใบอนุญาตประกอบวิชาชีพ"]`

**ผลกระทบ:** `/tags/`, Pagefind, ตัวกรองคำค้น

---

## draft

**ความหมาย:** ถ้า `true` หน้าจะไม่ publish ใน production build ปกติ

**ค่า production:** `false`

**ผลกระทบ:** **critical** ถ้า `true` ในข้อมูลที่จะ deploy

---

## weight

**ความหมาย:** ลำดับการแสดง (ใช้กับ `.md` ที่ Hugo เรียงเอง)

**ตัวอย่าง:** `100` (ค่าเริ่มต้นใน archetype)

**หมายเหตุ:** คลังจาก JSON เรียงด้วย logic ใน `laws-browser.js`

---

## og_image

**ความหมาย:** รูป Open Graph เฉพาะหน้า (ถ้ามี)

**ตัวอย่าง:** `/assets/feature-graphic.png` หรือว่างเพื่อใช้ค่า site default

**ผลกระทบ:** SEO / การแชร์ social

---

## ฟิลด์เสริมใน JSON

| ฟิลด์ | ความหมาย |
|--------|----------|
| `summary` | ย่อหน้าสรุปหลัก |
| `points` | bullet สาระสำคัญ |
| `citation` | ข้อความอ้างอิงสั้น |
| `links` | ลิงก์เพิ่มเติม `[{label, url}]` — ใช้ derive `source_url` ได้ |
| `relatedLaws` | id กฎหมายที่เกี่ยวข้องในคลัง |
