# เกมเรียนรู้ (Learning Games)

เอกสารนี้อธิบายส่วน **เกมเรียนรู้กฎหมายการศึกษา** สำหรับผู้ดูแลและผู้พัฒนาเว็บไซต์

## วัตถุประสงค์

เกมเรียนรู้เป็นกิจกรรมสั้น ๆ เพื่อช่วยให้ผู้ใช้ **ทบทวนคำสำคัญและแนวคิดพื้นฐาน** เกี่ยวกับกฎหมายการศึกษาอย่างผ่อนคลาย ไม่ใช่การสอบ ไม่ใช่การให้คำปรึกษากฎหมาย และไม่ใช่คำอธิบายกฎหมายฉบับสมบูรณ์

เกมแรกคือ **เกมจับคู่คำศัพท์กฎหมายการศึกษา** (matching/concentration) — เปิดการ์ดทีละสองใบเพื่อจับคู่ “คำสำคัญ” กับ “ความหมายเบื้องต้น”

## โครงสร้างไฟล์

| ไฟล์ | หน้าที่ |
|------|---------|
| `content/games/_index.md` | หน้า Landing ของเกม (`layout: list` → `layouts/games/list.html`) |
| `content/games/matching/_index.md` | หน้าเกมจับคู่ (`layout: matching` → `layouts/games/matching.html`) |
| `data/games/matching.yaml` | ข้อมูลคู่คำศัพท์ของเกมจับคู่ |
| `layouts/games/list.html` | เลย์เอาต์หน้า Landing |
| `layouts/games/matching.html` | เลย์เอาต์หน้าเกมจับคู่ (ฝัง JSON + อภิธานศัพท์สำหรับ Pagefind) |
| `layouts/partials/game-disclaimer.html` | ข้อความ disclaimer สำหรับหน้าเกม (ใช้ซ้ำได้) |
| `static/js/games-matching.js` | ตรรกะเกมจับคู่ (plain JS) |
| `static/css/style.css` | สไตล์ของเกม (บล็อก “เกมเรียนรู้ (Learning games)”) |
| `archetypes/games.md` | แม่แบบ front matter สำหรับหน้าเกมในอนาคต |

## ที่เก็บข้อมูลเกม

ข้อมูลเกมจับคู่อยู่ที่ `data/games/matching.yaml` และ Hugo อ่านเป็น `site.Data.games.matching`

### โครงสร้างของแต่ละรายการ

```yaml
items:
  - id: edu-law                 # รหัสไม่ซ้ำ (ใช้จับคู่ภายในเกม)
    term: "กฎหมายการศึกษา"        # คำสำคัญ
    match: "กฎหมายหรือเอกสาร..."  # ความหมายเบื้องต้น (กว้าง ปลอดภัย)
    explanation: "คำอธิบายสั้น..." # แสดงเมื่อจับคู่ถูก
    category: "พื้นฐาน"           # กลุ่มของคำศัพท์
    related_url: "laws/"          # ลิงก์อ่านต่อ (สัมพัทธ์กับ baseURL)
    related_label: "ไปที่คลังกฎหมาย"
```

## วิธีเพิ่มคู่คำศัพท์ใหม่

1. เปิด `data/games/matching.yaml`
2. เพิ่มรายการใหม่ใต้ `items:` โดยตั้ง `id` ที่ **ไม่ซ้ำกับรายการอื่น**
3. ใช้ถ้อยคำกว้าง ๆ ปลอดภัย (ดูหัวข้อถัดไป)
4. กำหนด `related_url` เป็นเส้นทาง **สัมพัทธ์กับ baseURL** เช่น `laws/`, `guides/`, `quiz/`, `about/source-policy/` — **ห้ามใส่** `/website-edulaw/` นำหน้า เพราะเทมเพลตจะแปลงให้อัตโนมัติด้วย partial `iu.html`
5. รัน `hugo server -D` แล้วเปิด `/games/matching/` เพื่อตรวจสอบ

> เกมจะสร้างการ์ดจำนวน `items × 2` ใบ จำนวนคู่ที่เหมาะสมเพื่อการแสดงผลบนมือถือคือประมาณ 6–10 คู่

## กติกาการใช้ถ้อยคำทางกฎหมายอย่างปลอดภัย

- ใช้คำ **กว้าง ๆ เชิงแนะนำ** ไม่ฟันธงข้อเท็จจริงทางกฎหมายเฉพาะกรณี
- ไม่สร้างข้อสรุปทางกฎหมายที่ละเอียดหรือเจาะจง
- ไม่ระบุว่าเป็นคำอธิบายกฎหมายที่ครบถ้วน
- ระบุเสมอว่าเป็น “ความหมายเบื้องต้น” หรือ “เพื่อการเรียนรู้”
- ทุกหน้าเกมต้องมี disclaimer (`game-disclaimer.html`) ที่ระบุว่าควรตรวจสอบแหล่งทางการก่อนใช้อ้างอิง

## วิธีเพิ่มเกมใหม่ในอนาคต

1. สร้างไฟล์ข้อมูล เช่น `data/games/<ชื่อเกม>.yaml`
2. สร้างหน้า `content/games/<ชื่อเกม>/_index.md` พร้อม front matter (ดู `archetypes/games.md`)
3. ตั้ง `layout:` ให้ตรงกับเลย์เอาต์ใหม่ เช่น `layout: <ชื่อเกม>` → `layouts/games/<ชื่อเกม>.html`
4. เพิ่มการ์ดลิงก์ในหน้า Landing `layouts/games/list.html`
5. เขียน JS เฉพาะเกมไว้ที่ `static/js/games-<ชื่อเกม>.js` และโหลดผ่านบล็อก `{{ define "scripts" }}`

## หมายเหตุด้านการเข้าถึง (Accessibility)

- การ์ดเป็น `<button>` กดได้ทั้งด้วยเมาส์ การแตะ และคีย์บอร์ด (Tab/Enter/Space)
- สถานะถูก/ผิดสื่อด้วย **ข้อความ** ไม่พึ่งสีอย่างเดียว (เครื่องหมาย ✓ และข้อความ “จับคู่แล้ว”)
- มี `:focus-visible` แสดงกรอบโฟกัสชัดเจน
- ผลตอบกลับและความคืบหน้าใช้ `aria-live` เพื่อให้โปรแกรมอ่านหน้าจอรับรู้
- ขนาดการ์ดบนมือถือใหญ่พอสำหรับการแตะ และเคารพ `prefers-reduced-motion`

## วิธีทดสอบในเครื่อง

```bash
hugo server -D                # เปิด /games/ และ /games/matching/
hugo --minify                 # ตรวจว่า build ผ่าน
npm run check:content         # ตรวจคุณภาพเนื้อหา (ไม่เกี่ยวกับเกมโดยตรง แต่ต้องไม่พัง)
npm run build:search          # สร้าง index Pagefind (อภิธานศัพท์ควรถูก index)
npm run report:stats          # รายงานสถิติ (ไม่บังคับ)
```

ตรวจด้วยตนเอง: การ์ดสับเปลี่ยน, จับคู่ถูกเปิดค้าง + แสดงคำอธิบาย, จับคู่ผิดคว่ำกลับ, ความคืบหน้าอัปเดต, ข้อความจบเกม, ปุ่ม “เล่นใหม่”, การแสดงผลบนมือถือ, คีย์บอร์ด, ไม่มี error ใน console
